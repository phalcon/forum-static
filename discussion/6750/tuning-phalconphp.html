Check this out: http://docs.phalcon.io/en/latest/reference/whats-next.html

Regarding Phalcon:

  • Cache resultsets to avoid hit the database, http://docs.phalcon.io/en/latest/reference/models-cache.html
  • Use the files or apc to store models meta-data, http://docs.phalcon.io/en/latest/reference/models.html#caching-meta-data
  • Use the files or apc to store annotations, http://docs.phalcon.io/en/latest/reference/annotations.html#annotations-adapters
  • Cache the output of view fragments when possible, http://docs.phalcon.io/en/latest/reference/views.html#caching-view-fragments
  • Implement HTTP cache when possible: http://docs.phalcon.io/en/latest/reference/response.html#cache-control
