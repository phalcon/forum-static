This syntax: case when used = 0 and team = 2 then points end is not supported by PHQL, you can use an IF:

IF(used = 0 and team = 2, points, 0)

The problem is not the CASE, it's that you're trying to concatenate using || which is a postgresql-only operator, you can:

  • Create a PHQL operator for that, check "Custom Dialect Functions" (https://blog.phalcon.io/post/phalcon-2-0-3-released)
  • Use the database component which gives you full freedom to use any postgresql extension losing retro-compatibility with other RBDMs https://docs.phalcon.io/en/latest/reference/db.html#finding-rows
