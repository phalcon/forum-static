Are you using raw SQL? Can you show us the full query?

After some cursory searching, it seems changing the collation on the fly disables indexes, so your query could really slow down. One common suggestion is to make a separate column in the table just for searching. When saving, both Joao and Jo達o get converted to Joao and stored in that special "search" column. This allows indexing to still work, greatly speeding up the query.
