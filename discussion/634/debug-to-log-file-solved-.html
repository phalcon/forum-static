I am not sure what you mean. But you can use a try catch with logger:

<?php
error_reporting(E_ALL);

try {
    /**
     * Read the configuration
     */
    $config = include __DIR__ . "/../app/config/config.php";

    /**
     * Read auto-loader
     */
    include __DIR__ . "/../app/config/loader.php";

    /**
     * Read services
     */
    include __DIR__ . "/../app/config/services.php";

    /**
     * Handle the request
     */
    $application = new \Phalcon\Mvc\Application($di);

    echo $application->handle()->getContent();

} catch (\Exception $e) {
     $logger = new \Phalcon\Logger\Adapter\File('/tmp/my-log.log');
     $logger->error($e->getMessage());
}

Depends on how and what do you want to log. See http://docs.phalcon.io/en/latest/reference/logging.html

You can use the logger component or a simple error_log would do the trick

} catch(\Exception $e) {
    $message = get_class($e), ": ", $e->getMessage(), "\n"
             . " File=", $e->getFile(), "\n"
             . " Line=", $e->getLine(), "\n"
             . $e->getTraceAsString() . "\n";
    error_log($message, 3, "/logs/current-date.log");
}

http://docs.phalcon.io/en/latest/reference/logging.html#file-logger
