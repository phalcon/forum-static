Yes. Time it takes to verify the hash greatly depends on the work factor parameter of Phalcon\Security::hash(). The work factor (aka cost) is none other than base-2 logarithm of the iteration count for the underlying Blowfish-based hashing algorithm (see http://php.net/crypt). Say, if the work factor is 20, the original password is iteratively hashed 2^20 times. This is used to make brute force attacks not feasible.
