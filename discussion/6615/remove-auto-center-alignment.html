Figured it out:

Turns out if you create a Controller (or scaffold using dev-tools which I did) titled users, then all templates in the users view directory will have <div align="center"> ... </div> wrapped around the loaded template through the {% raw %}{{{% endraw %} content() {% raw %}}}{% endraw %} function. Renaming the controller and the views directory to anything else will remove the align="center" element.

I have no idea how to debug Phalcon so I unfortunately cannot supply a patch fix for this.

when you create for example a scaffold for, lets say, 'access-list', phalcon will create:

-controller
 --AccessListController.php
-model
 --AccessList.php
-view
 --access_list
   ---access_list.volt
 --layouts
   ---access_list.volt

If you take a look in that layout you'll see

<div align="center">
    {% raw %}{{{% endraw %} content() {% raw %}}}{% endraw %}
</div>

remove the div and that's all. Problem solved. Also you can remove the layouts directory entirely if you don't need it
