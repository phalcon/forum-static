Don't think your assumption is correct on a result only being returned when multiple models are involved. The example only seems to show one: http://docs.phalcon.io/en/latest/api/Phalcon_Mvc_Model_Message.html.

Since getModel() returns the model that produced the message if there are two the model with the exception will be returned.
