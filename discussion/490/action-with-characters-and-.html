I found it. It looks like Phalcon 1.2.0 changed, (or always it works this way) controller action isn't camelized, only controller name is camelized, and @Firnis reply is correct. You can use eventManager to handle and calelize controller action. More about it is in @andresgutierrez reply here http://forum.phalcon.io/discussion/355/camelcase-actions
